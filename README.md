# olist-sql-analysis
Анализ датасета бразильского маркетплейса Olist с помощью PostgreSQL. 
Проект включает профилирование данных, выявление аномалий и аналитические запросы 
с использованием CTE и оконных функций.

## Используемые технологии
- PostgreSQL
- DBeaver

## Структура данных
Использованы таблицы: `customers`, `orders`, `order_items`, `products`, `payments`. 
Схема связей: `customers` → `orders` → `order_items` → `products`, а также `orders` → `payments`.

## Профилирование данных и находки

При проверке таблицы `orders` на пропуски было выявлено, что отсутствие дат доставки 
в большинстве случаев логично объясняется статусом заказа (заказ ещё не доставлен или отменён). 
Однако найдено две аномалии:

1. **6 заказов со статусом `canceled`, но с заполненной датой доставки.**
5 из них были доставлены значительно раньше обещанного срока, 1 заказ с опозданием. 
Возможно это и стало причинами отказов.

3. **8 заказов со статусом `delivered`, но без даты фактической доставки клиенту.** 
У 7 заказов дата получения не зафиксирована: вероятно, ошибка финального статуса. 
У 1 заказа отсутствуют обе даты (ни передачи перевозчику, ни доставки)

Также при анализе помесячной выручки выявлено, что период сентябрь–декабрь 2016 
и сентябрь–октябрь 2018 содержат единичные заказы, то есть представляют собой аномальные явления.
Поэтому эти периоды исключены из анализа.

## Аналитические запросы

### 1. Динамика выручки по месяцам
CTE + LAG() для расчёта изменения выручки относительно предыдущего месяца. 

**Вывод:** 

В 2017 году выручка растет на протяжении всего года, но есть просадки в апреле, июне, декабре.
Пик пришеося на ноябрь, минимальная выручка была в январе (вероятно это начало работы)

D 2018 году выручка растет, есть небольшие провалы в феврале, июне, августе.
Пик пришелся на апрель

### 2. Топ-3 категории товаров по выручке за весь период
Группировка по категории + SUM(price)

**Вывод:** лидирующие категории за весь период: 
`beleza_saude`, `relogios_presentes`, `cama_mesa_banho`

### 3. Месяцы с максимальным числом оплат по купонам.
Группировка по месяцу + сортировка по количеству платежей.

**Вывод:** пик оплат по купонам — январь 2018 и ноябрь 2017.

### 4. Клиенты с повторными покупками
Оконные функции MIN(), MAX() с PARTITION BY  для определения 
первой и последней покупки каждого клиента.

### 5. Доля выручки по типу оплаты
SUM() OVER () без партиционирования для расчёта процента от общей суммы.

**Вывод:**
Оплата кредитной картой приносит 78,34% всей выручки.
Наименьшая доля  у дебетовых карт.
